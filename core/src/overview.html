The main propose of this library is to define the concepts of {@link de.hu.gralog.structure GrALoG-Structures} and {@link de.hu.gralog.algorithm GrALoG-Algorithms} that can be used, displayed and / or executed by GrALoG. The library can also be seen as a starting point for Plugin-Developers that can use these concepts to build their own types of {@link de.hu.gralog.structure GrALoG-Structures} and {@link de.hu.gralog.algorithm GrALoG-Algorithms}.

Besides giving an introduction to the concepts underlying GrALoG this documentation therefore also tries to provide hints for Plugin-Developers on how to build their own Plugins for GrALoG. Therefore most of the package-, class- or method- (including this overview-) -JavaDoc-comments contain a section called Plugin-Developers, that explains how to use this package, class or method when developing your own Plugin.

Plugin-Developers

A Plugin in GrALoG basically consists of a definition of {@link de.hu.gralog.structure GrALoG-Structures} and {@link de.hu.gralog.algorithm GrALoG-Algorithms}. To define your own {@link de.hu.gralog.structure GrALoG-Structure} you have to subclass {@link de.hu.gralog.structure.StructureTypeInfo} ( please refer to the description of {@link de.hu.gralog.structure} on how to do this ). {@link de.hu.gralog.algorithm GrALoG-Algorithms} are basically classes that implement the {@link de.hu.gralog.algorithm.Algorithm}- interface, please refer to {@link de.hu.gralog.algorithm} on how to build your own algorithms.

Every Plugin in GrALoG comes as a Jar-File that must contain a file called plugin.config in its root-directory. The plugin.config tells GrALoG which GrALoG-Structures and -Algorithms are defined by that Plugin and where to lookup the classes that define them.

The plugin.config is an XML-File, that has the following format: 

	<plugin name="PLUGIN_NAME">
		<description>
			<![CDATA[PLUGIN-DESCRIPTION]]>
		</description>
		<structures>
			<structure typeInfoClass="TYPE_INFO_CLASS"/>
		</structures>
		<algorithms>
			<algorithm name="ALGORITH_NAME" class="ALGORITH_CLASS"/>
		</algorithms>
	</plugin>
Here PLUGIN_NAME, PLUGIN-DESCRIPTION, TYPE_INFO_CLASS, ALGORITHM_NAME and ALGORITHM_CLASS, have the following meaning:

Field Meaning
PLUGIN_NAME This is the name for your plugin, that is displayed to the User by GrALoG
PLUGIN_DESCRIPTION The description for this plugin enclosed in a CDATA-Element. Can be an arbitrary HTML-String
TYPE_INFO_CLASS The fully qualified classname for your subclass of {@link de.hu.gralog.structure.StructureTypeInfo}, that defines your {@link de.hu.gralog.structure GrALoG-Structure}
ALGORITHM_NAME This is the name for your {@link de.hu.gralog.algorithm GrALoG-Algorithm} that is displayed to the user by GrALoG
ALGORITHM_CLASS The fully qualified classname for your class implementing {@link de.hu.gralog.algorithm.Algorithm}, that defines your {@link de.hu.gralog.algorithm GrALoG-Algorithm}.
